# Code of Conduct

## Our Standard

NeepWords 希望建立一个直接、专业、可协作的开源环境。参与讨论、提 issue、提交 PR 时，请遵守以下原则：

- 以技术问题为中心，而不是针对个人
- 对事实、复现步骤和证据负责
- 尊重不同经验水平的贡献者
- 接受建设性反馈，也以建设性方式提出反馈
- 在讨论中避免人身攻击、羞辱、骚扰、歧视或挑衅性言论

## Unacceptable Behavior

以下行为不被接受：

- 人身攻击、侮辱、威胁或骚扰
- 基于身份、背景、地区、语言、性别、宗教等进行歧视
- 明知缺乏证据却反复进行指责
- 发布他人的隐私信息
- 故意刷屏、挑衅、破坏讨论秩序

## Maintainer Responsibilities

维护者有权也有责任：

- 设定并执行社区行为标准
- 删除不当评论、issue、PR 或其他内容
- 对不当行为发出警告、限制互动或拒绝协作

## Reporting

如果你遇到骚扰、歧视、恶意攻击或其他不当行为，请不要在公开 issue 中升级冲突。

建议通过以下方式私下联系维护者：

- GitHub 个人资料中的联系方式
- GitHub 私信或其他仓库维护者公开提供的私下渠道

维护者会尽量审查并处理报告，但处理结果会根据证据、上下文和社区安全需要决定。

## Scope

本行为准则适用于：

- 仓库 issue、PR、discussion
- 与本项目相关的代码审查和协作沟通
- 维护者代表本项目进行的公开互动

## Project Principle

NeepWords 更重视清晰、严谨、可复现的协作，而不是情绪化争论。请把讨论拉回到代码、数据、文档和可验证行为本身。
